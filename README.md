# kworkworks

Монитор публичной биржи проектов Kwork.

Схема работы:

1. GitHub Actions запускает `scraper.py`.
2. Playwright открывает публичную страницу `https://kwork.ru/projects?c=all`.
3. Скрипт читает карточки проектов прямо из DOM, включая скрытое полное описание, бюджет, заказчика, статистику, вложения и внешние ссылки.
4. Результаты сохраняются в `data/projects.json`; новые относительно предыдущего запуска — в `data/new_projects.json`.
5. Workflow коммитит обновлённые JSON обратно в репозиторий.

## Запуск

Workflow запускается:

- вручную из GitHub Actions (`workflow_dispatch`);
- по расписанию;
- при изменении файла `trigger.txt` — это позволяет запускать монитор из ChatGPT через подключённый GitHub.

По умолчанию сканируются первые 3 страницы. Переменная окружения `KWORK_PAGES` меняет глубину обхода.

Скрипт не авторизуется в Kwork, не обходит CAPTCHA и не пытается обходить технические ограничения сайта. При блокировке/изменении DOM он завершится с ошибкой.
